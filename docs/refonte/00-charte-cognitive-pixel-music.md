## 1. Idée directrice

**TEMPOSYSTEM IS ENERGY**

L'identité visuelle de TEMPOSYSTEM représente une énergie produite par la coopération humaine, conservée par la mémoire et orientée par les décisions collectives.

L'univers ne doit pas évoquer un logiciel administratif. Il doit évoquer simultanément : un instrument de musique ; un moteur énergétique ; un système cognitif ; un observatoire scientifique ; une constellation vivante ; une mémoire en mouvement.

Le langage visuel porte un nom : **COGNITIVE PIXEL MUSIC**.

Le pixel est l'unité minimale. Le rythme organise les pixels. La mémoire conserve leurs traces. La synchronisation les transforme en énergie collective.

## 2. Personnalité visuelle

TEMPOSYSTEM doit être : mystérieux, mais compréhensible ; technologique, mais humain ; expérimental, mais rigoureux ; spectaculaire, mais jamais bruyant ; abstrait, mais porteur de sens ; sombre, mais lumineux ; musical, mais silencieux par défaut ; futuriste, sans imiter le cyberpunk ; précis, sans devenir froid ; vivant, sans ressembler à un jeu vidéo.

**Trois mots essentiels** — *Rythme* : l'énergie n'est jamais immobile. *Mémoire* : chaque mouvement laisse une trace. *Orientation* : les signaux ne circulent pas au hasard.

## 3. Le symbole central — la spirale énergétique

La spirale TEMPOSYSTEM est le symbole fondamental du système. Elle représente : le temps ; l'accumulation ; la transmission ; la mémoire ; la circulation ; la transformation ; l'énergie qui grandit sans perdre son origine.

Elle ne doit pas être dessinée comme une galaxie décorative. Elle doit sembler composée de : pixels ; impulsions ; fragments lumineux ; données rythmiques ; particules synchronisées.

**Quatre états** : 1 — *Signal* (quelques pixels isolés pulsent) · 2 — *Synchronisation* (les pixels trouvent un mouvement commun) · 3 — *Énergie* (ils composent une spirale lumineuse) · 4 — *Rayonnement* (la spirale transmet des impulsions vers d'autres éléments).

## 5. Logo

**Construction** : spirale énergétique + mot TEMPOSYSTEM + éventuellement la signature IS ENERGY.
- *Version horizontale* : spirale à gauche, TEMPOSYSTEM à droite, IS ENERGY en dessous ou aligné en taille secondaire.
- *Version compacte* : spirale seule (favicon, icône d'app, avatar, bouton du Conseil, indicateur de chargement, pulsation du système).
- *Version instrument* : le symbole inscrit dans un cercle, une matrice carrée, une grille de pixels ou une orbite fine — pour les interfaces techniques.

**Zone de protection** : espace minimal équivalent au diamètre du cœur de la spirale. Aucun texte, trait, bord ou particule ne pénètre cette zone.

**Interdictions** : ombre volumétrique classique ; effet chromé ; logo en bouton 3D ; dégradé arc-en-ciel permanent ; spirale systématiquement enfermée dans un cercle ; déformation ; multiplication des contours ; logo utilisé comme décoration de fond.

## 8. Règles de couleur

Une section n'utilise jamais toutes les couleurs simultanément. Règle générale : une dominante + une secondaire + une couleur d'impulsion + le blanc énergétique + le fond sombre.

- *Section Mémoire* : bleu dominant, violet secondaire, blanc pour les traces consolidées, cyan pour les nouvelles informations.
- *Section Conseil* : violet dominant, orange pour les décisions, magenta pour les contributions humaines, vert pour les validations.
- *Section Écosystème* : orange pour la source énergétique, cyan et bleu pour les transmissions, couleurs propres aux applications en accents limités.

Les dégradés expriment une transformation réelle (ex. `Magenta → Violet → Bleu` = Contribution → Synchronisation → Mémoire). Jamais de dégradé purement esthétique.

- **A. Voix principale** (textes, navigation, descriptions, interfaces, boutons, documentation) : sans serif contemporaine, géométrique, très lisible. Choix recommandé : **Space Grotesk**.
- **B. Voix pixel** (valeurs, impulsions, séquences, état du système, coordonnées, équation, micro-titres techniques, mots-clés, signaux) : typographie pixel/bitmap sobre — jamais de longs paragraphes. Court terme : **Geist Pixel ou Departure Mono**. Long terme : **TEMPO Pixel** (police propriétaire sur grille de 8 pixels).

## 10. Hiérarchie typographique

- *Titre manifeste* : très grand, peu de mots (« TEMPOSYSTEM IS ENERGY »).
- *Titre de section* : clair, éditorial, 4 à 8 mots (« La mémoire stabilise l'énergie. »).
- *Label système* : petites capitales, espacement généreux (MÉMOIRE ACTIVE · SIGNAL ENTRANT · ADR-0010).
- *Donnée énergétique* : pixel ou monospace (+1 MANA · 00:00:01 · SYNCHRONIZED).
- *Corps de texte* : largeur contenue, 60–70 caractères par ligne max.

## 11. La grille

- **Desktop** : 12 colonnes, contenu max **1440 px**, marges généreuses. Les illustrations peuvent dépasser la grille (sensation d'expansion).
- **Tablette** : 8 colonnes, visualisations simplifiées.
- **Mobile** : 4 colonnes. Les interfaces orbitales deviennent séquences verticales, anneaux partiels, carrousels ou compositions focalisées. Ne jamais réduire mécaniquement une constellation desktop.

## 12. Formes — vocabulaire limité

Le pixel (unité minimale) · le point lumineux (signal, présence) · la ligne (transmission) · l'onde (rythme, propagation) · l'orbite (fonction, relation) · la spirale (mémoire, énergie) · la grille (structure) · le cercle incomplet (processus actif) · le halo (rayonnement) · le spectre (intensité, activité). Éviter les formes décoratives sans fonction.

## 13. Style pixel

Fin ; dense à certains endroits ; presque invisible à d'autres ; précis ; lumineux ; abstrait ; contemporain.

**Échelles** : *micro-pixel* 1–2 px (données, poussières, traces) · *pixel signal* 3–5 px (impulsions, circulations, interactions) · *pixel symbolique* 8–16 px (icônes, fonctions, objets visuels) · *macro-pixel* exceptionnel (titres, transitions, moments manifestes, révélations).

**Règle essentielle** : la pixellisation doit être nette. Pas de flou général. Les halos peuvent entourer les pixels, leur cœur reste précis.

## 14. Iconographie

Grille commune **16×16 ou 24×24 px**. Contour pixel, géométrie simple, un symbole central, animation limitée, lisibilité immédiate.

Exemples : Mémoire = fragment/livre de pixels consolidés · Vision = étoile polaire/cap lumineux · Cohérence = deux rythmes qui se synchronisent · Construction = assemblage de blocs · Protection = bouclier incomplet qui se ferme · Anticipation = onde qui atteint l'horizon · Communication = signal transmis entre deux points · Éthique = balance abstraite/cercle de protection.

**Éviter les emojis** comme langage visuel principal du site public (tolérés dans le Livre de Bord et les espaces narratifs).

## 15. Mouvement

Le mouvement est une composante identitaire, pas un effet ajouté. Tempo général : lent, précis, hypnotique. La page doit sembler respirer.

- **Pulsation fondamentale** : cycle ≈ 4 s (la spirale/le cœur inspire et expire légèrement).
- **Seconde contributive** : un `+1` discret peut apparaître chaque seconde — signal symbolique du passage du temps, jamais un compteur mensonger ; caractère conceptuel indiqué si nécessaire.
- **Transmission** : impulsion d'un point vers un autre, 600–1200 ms.
- **Synchronisation** : éléments désordonnés trouvant un rythme commun, 2–4 s.
- **Décision** : un signal converge, devient orange, puis rayonne.
- **Mémoire** : une impulsion mobile ralentit, devient bleue et rejoint une trace persistante.

## 16. Intensité du mouvement — trois niveaux

- **Niveau 0 — Statique** : `prefers-reduced-motion`, appareils modestes, captures, documents, économie d'énergie.
- **Niveau 1 — Respiration** *(défaut)* : halos, oscillation légère, pulsations, quelques particules.
- **Niveau 2 — Immersion** : réactions au pointeur, propagation des signaux, parallaxe légère, visualisation audio, interaction avec le Conseil. Activé volontairement.

## 17. Pixel music — signatures rythmiques des fonctions

Chaque fonction permanente possède une signature rythmique (d'abord visuelle ; couche sonore optionnelle ultérieure) :

Capitaine = battement central lent et stable · Timonier = rythme régulier qui réaligne les autres · Charpentier = séquence progressive et structurée · Calfat = impulsions courtes de contrôle et correction · Enlumineur = variations chromatiques et harmoniques · Cartographe = traces longues et connexions entre événements · Vigie = signaux espacés provenant de l'horizon · Gardien = pulsation grave, rare et protectrice · Messager = signaux rapides circulant entre les orbites · Quartier-Maître = accord de synthèse reliant plusieurs signatures.

## 18. Son

Silencieux par défaut. Contrôle explicite : **« Écouter TEMPOSYSTEM »**. Son minimal, synthétique, doux, spatial, génératif, non intrusif. Éviter : musique de fond continue, basses agressives, sons arcade, bruitages au survol, démarrage automatique, volume élevé. L'activation sonore peut transformer les signaux visuels en notes.

## 19. Composants d'interface

- **Bouton principal** : fond lumineux maîtrisé, contour net, capitales modérées/petites capitales, impulsion traversante au survol (« ENTRER DANS LE CONSEIL → »).
- **Bouton secondaire** : transparent, bordure fine, lueur au survol/focus uniquement.
- **Carte système** : fond bleu nuit, bordure fine, rayon modéré, très peu d'ombre, ligne énergétique possible sur un bord, contenu fortement hiérarchisé.
- **Carte manifeste** : plus ouverte, pas nécessairement de bordure, texte et illustration respirent ensemble.
- **Donnée dynamique** : toujours accompagnée de son nom, sa source, son statut, sa date/son mode de calcul. **Les simulations sont identifiées comme telles.**

## 20. États système

Opérationnel = vert discret · En synchronisation = cyan/violet animé · En construction = orange doux · Prototype = jaune · En attente = bleu-gris · Incident = **rouge réservé exclusivement aux erreurs réelles** (jamais décoratif).

## 21. Photographie

Priorité à l'abstraction. Photos rares. Présence humaine : silhouettes, mains, mouvements, traces, foules-constellations, territoires-cartes énergétiques. Jamais de photos corporate génériques.

## 22. Familles d'illustrations

- **A. Energy Core** — spirale/étoile/cœur central → HERO, moteur, TEMPOSYSTEM IS ENERGY.
- **B. Signal Flow** — impulsions traversant une grille → communication, orchestration, API, circulation.
- **C. Cognitive Orbit** — fonctions permanentes autour d'un cœur → Conseil, équipage, architecture cognitive.
- **D. Memory Trace** — signaux mobiles devenant traces persistantes → ADR, mémoire, Livre de Bord.
- **E. Decision Pulse** — signaux convergents produisant une onde → consensus, arbitrage, gouvernance.
- **F. Ecosystem Transmission** — une source alimentant différentes infrastructures → MANAtimebank, MANA, projets futurs.
- **G. Time Sequence** — battements/pixels s'accumulant au fil des secondes → équation, temps contributif, transitions.

## 23. Niveaux de densité

**Silence** (beaucoup d'espace sombre, une phrase, quelques pixels) · **Rythme** (une séquence visuelle mesurée) · **Intensité** (grande illustration centrale, flux, constellation, révélation). Deux sections très intenses ne se suivent jamais. La page respire comme un morceau de musique.

## 24. Responsive

Desktop = expérience orchestrale complète. Mobile = expérience solo : un seul signal/une seule fonction à la fois, narration verticale. Les essentiels demeurent : la spirale, le rythme, l'équation, les fonctions, le Conseil, la mémoire. **Le mobile n'est pas une version appauvrie : c'est une mise en scène différente et intentionnelle.**

## 25. Accessibilité

La beauté ne dépend jamais exclusivement de la couleur, du mouvement, du son, d'un contraste faible ou d'un texte minuscule. Prévoir : contraste WCAG, focus clavier visible, textes alternatifs, descriptions des visualisations, désactivation du son, réduction du mouvement, pause des animations, données accessibles en texte, états accompagnés d'un libellé.

## 26. Ce que TEMPOSYSTEM ne doit jamais devenir

Ni plateforme crypto, ni casino numérique, ni interface de trading, ni jeu arcade rétro, ni terminal de hacker, ni landing page d'IA générique, ni dashboard SaaS de statistiques fictives, ni écran de contrôle militaire, ni univers cyberpunk dystopique, ni collection d'effets néon sans signification. **La technologie apparaît comme une force de coopération, pas comme un instrument de domination.**

## 27. Principe de composition — cinq questions par section

1. Quelle énergie entre dans cette section ?
2. Quelle transformation s'y produit ?
3. Quelle trace reste après cette transformation ?
4. Quelle énergie ressort de la section ?
5. Comment cette énergie nourrit-elle la suivante ?

Une section sans transformation visible ou narrative est simplifiée ou supprimée.

## 28. Promesse visuelle

À l'entrée : *« Quelque chose vit ici. »* — En poursuivant : *« Cette énergie possède une structure. »* — Puis : *« Cette structure conserve la mémoire. »* — Enfin : *« Je peux entrer dans ce système et contribuer à son mouvement. »*
